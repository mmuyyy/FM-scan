# 工业级FM信号处理系统

## 项目概述
工业级FM信号搜集、标注、解调、还原与可视化系统，用于实时扫描、检测、解调和解码FM广播信号。

## 功能特点

### 核心功能
1. **多频段扫描**：支持同时扫描FM广播频段(87.5-108 MHz)和5.8G FPV频段(5725-5925 MHz)
2. **信号检测**：基于带宽、SNR分析检测FM信号
3. **FM解调**：使用相位差分法进行FM解调
4. **信号分类**：自动识别VOICE、MUSIC、AUDIO等信号类型
5. **音频还原**：支持音频去加重、滤波和重采样
6. **实时可视化**：实时显示频谱、时域信号、扫描进度和检测到的信号
7. **数据存储**：自动保存检测到的信号信息、音频文件和元数据
8. **系统监控**：实时监控系统状态和错误处理

### 工业级特性
1. **高可靠性**：多路径API支持，确保兼容不同版本的UHD
2. **错误处理**：完善的错误处理和故障恢复机制
3. **配置管理**：灵活的配置系统，支持自定义参数
4. **资源管理**：高效的内存管理和资源使用
5. **24/7运行**：支持持续扫描模式，适合长时间运行
6. **详细日志**：完整的日志记录，便于故障排查
7. **模块化设计**：清晰的代码结构，便于维护和扩展

## 技术栈
- Python 3.8-3.11
- UHD (Universal Hardware Driver)
- NumPy
- SciPy
- Matplotlib
- PyAudio (可选，用于音频输出)

## 硬件要求
- USRP B210或B200mini
- 合适的天线（FM频段和5.8G频段）
- 计算机运行Windows或Linux

## 安装指南

### 1. 安装依赖

#### Windows
```bash
# 安装Python 3.8+ (推荐3.10)
# 安装UHD驱动
# 安装Python包
pip install numpy scipy matplotlib pyaudio
```

#### Linux
```bash
# 安装UHD驱动
sudo apt-get install libuhd-dev uhd-host

# 安装Python包
pip install numpy scipy matplotlib pyaudio
```

### 2. 配置USRP设备
- 连接USRP设备到计算机
- 确保UHD驱动能够正确识别设备
- 运行 `uhd_usrp_probe` 验证设备连接

## 使用方法

### 1. 基本使用
```bash
# 运行工业级FM信号处理系统
python fm_industrial_processor.py
```

### 2. 配置参数
系统会自动创建 `fm_processor_config.json` 配置文件，您可以根据需要修改以下参数：

```json
{
  "device": {
    "sample_rate": 2000000,      # 采样率 (Hz)
    "gain": 30,                  # 增益 (dB)
    "antenna": "RX2"             # 天线选择
  },
  "scan": {
    "bands": [
      {
        "name": "FM Broadcast",  # 频段名称
        "start": 87500000,        # 起始频率 (Hz)
        "stop": 108000000,        # 结束频率 (Hz)
        "step": 100000,           # 步进频率 (Hz)
        "color": "blue"           # 图表颜色
      },
      {
        "name": "5.8G FPV",
        "start": 5725000000,
        "stop": 5925000000,
        "step": 1000000,
        "color": "red"
      }
    ],
    "signal_detection": {
      "snr_threshold": 10,       # 信噪比阈值
      "bandwidth_min": 50000,    # 最小带宽
      "bandwidth_max": 300000    # 最大带宽
    }
  },
  "demodulation": {
    "deemphasis_tau": 7.5e-05,   # 去加重时间常数
    "audio_cutoff": 15000,       # 音频截止频率
    "audio_sample_rate": 48000   # 音频采样率
  },
  "visualization": {
    "update_interval": 50,       # 更新间隔 (ms)
    "fft_size": 1024,            # FFT大小
    "window_size": 50,           # 显示窗口大小
    "figsize": [16, 12]          # 图表大小
  },
  "output": {
    "save_audio": true,          # 保存音频
    "save_metadata": true,       # 保存元数据
    "output_dir": "output"       # 输出目录
  }
}
```

## 系统架构

### 模块结构
1. **ConfigManager**：配置管理模块，负责加载和保存配置
2. **SignalProcessor**：信号处理模块，负责信号分析、解调和分类
3. **USRP_Device**：设备管理模块，负责与USRP设备通信
4. **SignalVisualizer**：可视化模块，负责实时显示信号特征
5. **FM_Industrial_Processor**：主处理器，协调各个模块的工作

### 数据流
1. **设备初始化**：配置USRP设备参数
2. **数据采集**：从USRP接收IQ样本
3. **信号处理**：计算带宽、SNR和信号分类
4. **信号检测**：识别FM信号
5. **解调**：对检测到的信号进行FM解调
6. **音频还原**：处理解调后的音频信号
7. **数据存储**：保存信号信息和音频文件
8. **可视化**：实时显示信号特征
9. **系统监控**：监控系统状态和错误

## 故障排查

### 常见问题

1. **设备初始化失败**
   - 检查USRP设备连接
   - 验证UHD驱动版本
   - 检查权限设置

2. **数据接收失败**
   - 检查天线连接
   - 调整增益设置
   - 检查采样率设置

3. **信号检测不准确**
   - 调整SNR阈值设置
   - 检查信号源强度
   - 调整扫描步进频率

4. **可视化卡顿**
   - 降低FFT大小
   - 增加更新间隔
   - 关闭不必要的应用程序

5. **音频还原失败**
   - 检查PyAudio安装
   - 调整音频参数设置
   - 确保有足够的系统资源

## 输出文件说明

### 1. 音频文件
- 格式：WAV
- 路径：`output/audio_{frequency}MHz_{timestamp}.wav`
- 采样率：48kHz
- 位深：16位

### 2. 信号元数据
- 格式：JSON
- 路径：`output/signal_metadata_{timestamp}.json`
- 内容：包含频率、带宽、SNR、信号类型等信息
